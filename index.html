Front End Roles

Front-End Roles

Responsive Design

- Web pages automatically adjusts their layout, design and visual elements according to screen size and resolution of the device
- For example: Smartphones, tablets, desktops

CSS uses fluid images, flexible grids and media queries for this

Cross-Browser Compatibility

- Websites deliver consistent experience across different web browsers
- Form Elements, Fonts and Scroll bars render differently in firefox,chrome,microsoft edge and safari

Doctype,CSS reset, Responsive websites are best practices for above issues

Interactive Features

- Response to user actions which allows them to participate actively
- Dynamic and Engaging interfaces using animations, live updates, micro-interactions and personalized content

Hover effects, Accordions and Toggles, Interactive Videos and Slideshows, Voice AI and Chatbots are examples of interactive features

Web Server Content Delivery

Web Server Content Delivery

Client Request

- User enter URL in browser
- Browser sends an HTTP/HTTPS request to web server

Common request methods include GET to fetch resources and POST to send data to the server

DNS Resolution and Connection Establishment

- Browser contacts a DNS server to translate the URL i.e Domain Name into IP Address of the server
- Browser establishes a connection with the web server using this IP Address

It is done via TCP, optionally TLS in HTTPS

Request Processing

- Web Server receives the request
- Locates the requested file and may interact with the backend services

Static Servers

- Delivers pre-built web pages stored on the server as it is
- Pages created using HTML,CSS,JS languages

Dynamic Servers

- Generates content using databases and application servers based on real time
- Uses server side scripting languages lie PHP, Node.js or ASP.NET to process data and interact with databases

Sends HTTP/HTTPS Response with the requested content

Response Delivery

- Server sends back an HTTP/HTTPS response containing status code and requested content
- A three digit code indicating the result of a client's request

200 OK, 404 Not Found, 500 Internal Server Error etc

Rendering

- The browser parses the HTML and CSS files
- Displays the web page to the user

Constructs DOM tree, CSSOM tree and Render Tree for rendering web pages

Web Browser Content Rendering

Web-Browser Content Rendering

HTML Loading and Parsing

- Browser fetches html files and tokenizes it
- It constructs DOM tree

DOM-Document Object Model, which represents the structure of the document

CSS Loading and Parsing

- CSS files are fetched and parsed to create CSSOM tree
Parsing involves breaking down input data into smaller components called token and organizing them into structured format such as parse tree

CSSOM-CSS Object Model, which defines the styles for the elements in the DOM

Building Render Tree

- Browser combines DOM tree and CSSOM tree to build Render Tree
- It only includes visible elements and their computed styles

Non-visible elements such as those with display:none are excluded from this tree

Layout Stage

- The browser calculates exact position and size of each element in render tree
- It ensures that elements are aligned and sized properly

Follows layout rules such as CSS box model

Painting Stage

- Render tree is converted into actual pixels on the screen
- Elements are painted in specific order

Usually from back to front, and may split into layers for optimization

Composing Stage

- It combines above layes to produce final image displayed to the user
- Modern browsers often use GPU acceleration to enchance its performance

JavaScript execution pauses DOM and CSSOM construction until its execution as it can modify both

HTML

- Stands for Hyper Text Markup Language
- It creates the structure of the webpage

Basics of HTML

  1. Document Type

    2. Tells the version of HTML i.e HTML5, which is current version

  2. HTML Root Tag

    3. It is the root element of HTML Document.everything inside it is considered a part of the HTML document

  3. Head Tag

    4. It contains metadata about the document and is not displayed on screens

    • Meta Tag

      • Charset and viewport are its attributes

    • Title Tag

      • It appears in the browser tab

    • Link Tag

      • It is used to link external files such as css file using its rel and href attributes

  4. Body Tag

    5. It contains the actual content of the webpage that will be visible to user

    • Headings

      • Defines hierarchy of headings from h1 to h6 in a web page

    • Paragraphs

      • Used to add paragraphs to the web page such as explanation of a heading

    • Images

      • The img tag is used to add images to a web page

    • Tables

      • Following tb,th,td tags are used to define a table,columns and rows of table respectively

    • Div

      • It is used to group different elements to form a section, mainly used for layout and styling

    • Scripts

      • Script tag is used to add JavaScript code to the elements to enhance interactivity

    • Lists

      • The ul,ol,li tags are used to created unordered,ordered lists respectively

    • Links

      • The a tag is used to create links for navigation on page,redirection to another page

Best Practices
  • Use Semantic HTML
  • Include Alternative text for Images
  • Minimize Inline styles and scripts

CSS

- Stands for Cascading Style Sheets
- It designs the webpages to look asthetic

Basics of CSS

  1. Syntax

    2. Rules to follow while writing a css file so that the browser can interpret it correctly

    h1{color:red;} - h1 selector,color property,red value, this whole thing is syntax for styling every elemtent

  2. Selectors

    3. Used to find elements to style them

    Simple Selectors Name,Id,Class | Combinator Selectors | Pseudo-class Selectors | Pseudo-element Selectors | Attribute Selectors

  3. Properties

    4. It specifies style attribute such as "color" for text color

  4. Values

    5. Defines the property's setting such as "red" for the text color

  5. Box Model

    6. Defines how elements are sized and positioned by assinging a box in the DOM tree that determines an element's dimensions and its position relative to other elements

    • Content

      • The part that hold actual content of an element

    • Padding

      • Space around an element's content, inside of defined borders

    • Border

      • Specify the style,width and color of an element's border

    • Margin

      • Space around an element's content, outside of defined borders
